# Мониторинг ПК — телеметрия на OLED

Режим **мониторинга с ПК** выводит на дисплей Heltec V4 телеметрию компьютера: CPU/GPU, RAM, диски, медиа, вентиляторы, материнская плата, погода. Данные передаются по TCP с сервера на ПК (Python + Libre Hardware Monitor).

---

## Назначение

- Отображение в реальном времени на OLED 128×64: загрузка и температура CPU/GPU, использование RAM, температуры дисков, статус воспроизведения, обороты вентиляторов, данные по материнской плате, погода.
- Несколько сцен (экранов), переключаемых коротким нажатием кнопки.
- Алерты при превышении порогов (RED ALERT, мигание LED, подсветка метрики на экране).

---

## Требования

- **Устройство:** Heltec WiFi LoRa 32 V4 (ESP32-S3) с прошивкой Nocturne.
- **ПК:** Windows с установленным [Libre Hardware Monitor](https://github.com/LibreHardwareMonitor/LibreHardwareMonitor/releases) (запуск **от имени администратора**, в настройках включён **Remote Web Server**, порт 8085).
- **Сеть:** Устройство и ПК в одной Wi‑Fi сети; в `secrets.h` указаны SSID, пароль и IP ПК.
- **Сервер на ПК:** Python 3.x и зависимости из `server/requirements.txt` (или готовый exe из `server/build_server.bat`).

---

## Установка сервера на ПК

1. Клонировать репозиторий, открыть проект в PlatformIO/VS Code.
2. Установить зависимости:
   ```bash
   cd server
   pip install -r requirements.txt
   ```
3. Настроить `server/config.json`:
   - `host`: обычно `"0.0.0.0"` (слушать все интерфейсы).
   - `port`: TCP-порт (по умолчанию **8888**). Должен совпадать с `TCP_PORT` в `include/secrets.h` на устройстве.
   - `lhm_url`: адрес LHM, обычно `"http://localhost:8085/data.json"`.
   - `limits`: пороги для алертов (cpu, gpu в °C или % — см. логику в `monitor.py`).
   - `weather_city`: город для погоды (опционально).
4. Запуск сервера:
   ```bash
   cd server
   python monitor.py
   ```
   Варианты: с треем (`--no-tray` — без иконки в трее), в консоль (`--console`). Логи пишутся в `nocturne.log` (в корне проекта или рядом с exe при сборке).
5. Сборка exe (опционально): из папки `server` запустить `build_server.bat`. Результат: `server/dist/NocturneServer.exe`.

---

## Настройка устройства

1. Скопировать `include/secrets.h.example` в `include/secrets.h`.
2. Заполнить в `secrets.h`:
   - `WIFI_SSID` — имя Wi‑Fi сети.
   - `WIFI_PASS` — пароль сети.
   - `PC_IP` — IP-адрес ПК в локальной сети (где запущен `monitor.py`).
   - `TCP_PORT` — порт (тот же, что в `server/config.json`, например 8888).
3. Собрать и загрузить прошивку (PlatformIO: env `heltec_wifi_lora_32_V4`).
4. Включить устройство: оно подключится к Wi‑Fi и установит TCP-соединение с ПК. Если сервер запущен, на экране появятся сцены с данными.

---

## Запуск

1. Запустить **Libre Hardware Monitor** от имени администратора, включить Remote Web Server (порт 8085).
2. Запустить сервер телеметрии: `python server/monitor.py` (или exe из `server/dist/`).
3. Включить устройство или перейти в режим мониторинга из меню (меню → **Monitoring** → **PC**). После подключения к Wi‑Fi и TCP устройство начнёт получать JSON и отображать сцены.

---

## Сцены (экраны)

В режиме мониторинга короткое нажатие кнопки переключает сцены по кругу:

| № | Сцена    | Содержимое |
|---|----------|------------|
| 0 | **MAIN** | CPU/GPU темпы и загрузка, RAM, общая полоса (lifebar) |
| 1 | **CPU**  | Ядро, температура, частота, нагрузка, мощность |
| 2 | **GPU**  | Температура, частота, нагрузка, VRAM |
| 3 | **RAM**  | Топ процессов по памяти, общее использование |
| 4 | **DISKS**| Сетка 2×2: буква диска, температура |
| 5 | **MEDIA**| Трек/исполнитель, статус воспроизведения |
| 6 | **FANS** | RPM и % для CPU, Pump, GPU, Case |
| 7 | **MB**   | Материнская плата: VRM, Chipset и др. |
| 8 | **WEATHER** | Иконка погоды (XBM), температура |

## Алерты (RED ALERT)

Сервер при превышении порогов отправляет пакет с `alert: "CRITICAL"` и указанием метрики. Устройство:

- мигает белым LED (GPIO 35) два раза;
- переключает экран на сцену с проблемной метрикой (CPU/GPU/RAM);
- подсвечивает метрику на экране и рисует RED ALERT по краям.

Пороги задаются в `server/config.json` (и дублируются для справки в `include/nocturne/config.h`). Метрики: CPU temp, GPU temp, CPU load, GPU load, VRAM load, RAM (использовано ГБ). Гистерезис (сброс алерта) настраивается в `server/monitor.py`.

## Устранение неполадок

| Проблема | Что проверить |
|----------|----------------|
| Нет связи (NO SIGNAL, NET:--) | Wi‑Fi: SSID/пароль в `secrets.h`, устройство в той же сети. TCP: IP ПК и порт в `secrets.h` совпадают с `config.json`; сервер запущен; firewall не блокирует порт. |
| Нет данных по дискам | LHM: в настройках включены диски; в `server/monitor.py` или `config.json` указаны нужные сенсоры. Утилита `server/tools/dump_lhm_disks.py` помогает посмотреть структуру data.json. |
| Медиа не обновляется | Windows: установлен пакет `winsdk` (для GlobalSystemMediaTransportControls). Сервер должен быть запущен с поддержкой медиа (см. код monitor.py). |
| Алерты не срабатывают | Пороги в `config.json` (limits); в логах сервера — отправка alert; на устройстве — сцена переключается по alert_metric. |

Подробнее: комментарии в `server/monitor.py`, утилиты в `server/tools/` (например, `dump_lhm_disks.py` для отладки дисков).
