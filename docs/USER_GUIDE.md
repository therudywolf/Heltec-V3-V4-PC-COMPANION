## 1. Быстрый старт

1. **Прошивка:** скопируйте `include/secrets.h.example` в `include/secrets.h`, укажите Wi‑Fi (SSID, пароль) и при необходимости IP ПК и порт. Соберите проект в PlatformIO (env `heltec_wifi_lora_32_V4`) и загрузите прошивку через USB.
2. **Первый запуск:** после загрузки появится заставка, затем экран мониторинга (если Wi‑Fi подключён и сервер на ПК запущен — появятся данные). Если сервера нет — будет «NO SIGNAL» или «NET:--».
3. **Меню:** двойной тап по кнопке (PRG, GPIO 0) открывает главное меню. Короткое нажатие — следующий пункт, долгое — вход в категорию или выполнение действия.
4. **Выход из меню:** в подменю — двойной тап возвращает на уровень выше; на уровне категорий двойной тап закрывает меню.

Подробная установка: [README.md](../README.md#установка), [HELTEC_V4_BOARD_AND_CONFIG.md](board/HELTEC_V4_BOARD_AND_CONFIG.md).

## 2. Кнопка и жесты

Одна кнопка **PRG (GPIO 0)** управляет всем интерфейсом.

| Жест | Длительность / условие | Действие (вне меню) | В меню |
|------|------------------------|----------------------|--------|
| **Короткое** | \< 1 с | Следующая сцена (в мониторинге) или следующий пункт | Следующий пункт списка |
| **Долгое** | 1–2.5 с | Переключение яркости (норм./пониж.) | Вход в подменю или выполнение действия |
| **Очень долгое (Predator)** | ~2.5 с | Экран гаснет, LED «дышит»; повтор — выход | — |
| **Двойной тап** | Два быстрых нажатия | Открыть меню | Назад / закрыть меню |

Во всех полноэкранных режимах внизу отображается подсказка, например: **1x next  2s ok  2x menu** (короткое — следующий пункт, долгое — OK, двойной тап — меню/назад).

## 3. Меню: навигация и выход

- **Открытие меню:** в любом режиме (кроме глубокого сна) — **двойной тап**.
- **Категории (уровень 0):** Monitoring, Hacker, BMW, Config, System. Короткое нажатие — переход к следующей категории, долгое — вход в выбранную.
- **Подменю (уровень 1):** список пунктов. Короткое — следующий пункт, долгое — выполнить (перейти в режим или применить настройку).
- **Выход:** двойной тап из подменю — назад к категориям; с уровня категорий — закрыть меню и вернуться в текущий режим.

Подсказка внизу: `1x next  2s ok  2x menu`.

## 4. Режим мониторинга ПК

**Вход:** Меню → **Monitoring** → **PC** (долгое нажатие).

- На экране отображаются сцены с телеметрией с компьютера (CPU, GPU, RAM, диски, медиа, вентиляторы, погода). Данные приходят по TCP с сервера на ПК (`server/monitor.py`).
- **Короткое нажатие** — переключение сцен по кругу (MAIN → CPU → GPU → RAM → DISKS → MEDIA → FANS → MB → WEATHER → MAIN).
- Для работы нужны: Wi‑Fi в `secrets.h`, запущенный Libre Hardware Monitor (Remote Web Server 8085), запущенный `python server/monitor.py` (порт в `config.json` совпадает с `TCP_PORT` в `secrets.h`).

Подробно: [PC_MONITORING.md](monitoring/PC_MONITORING.md).

## 5. Режим зарядки (Charge only)

**Вход:** Меню → **System** → **Charge only** (долгое нажатие).

- Экран показывает заряд батареи в том же стиле, что и главный экран мониторинга: блок **BAT** (процент), блок **V** (напряжение), нижняя строка — статус **CHARGING** / **READY** и напряжение. При отсутствии батареи — «N/A» и «Connect USB».
- Wi‑Fi отключается для экономии энергии. Выход: двойной тап → меню → двойной тап (закрыть меню).

## 6. Режим BMW Assistant

**Вход:** Меню → **BMW** → **BMW Assistant** (долгое нажатие).

### Кратко

- **BLE-ключ:** устройство рекламируется как «BMW E39 Key». Подключение телефона → разблокировка по I-Bus; отключение (через ~2.5 с) → блокировка.
- **Управление с платы:** на экране отображается текущее выбранное действие. **Короткое нажатие** — следующее действие в списке (Goodbye, FollowMe, Park, Hazard, LowBeam, LightsOff, Unlock, Lock, Trunk, Cluster, DoorUnlk, DoorLock). **Долгое нажатие** — выполнить выбранную команду по I-Bus (при отсутствии связи с шиной — «No IBus»).
- **Выход:** двойной тап → меню → с уровня категорий снова двойной тап.

### Управление светом и замками с телефона (BLE)

При подключении к «BMW E39 Key» можно отправлять команды по BLE (GATT-характеристика):

- Запись **одного байта** (0–11) выполняет соответствующую команду: 0 = GoodbyeLights, 1 = FollowMeHome, 2 = ParkLights, 3 = HazardLights, 4 = LowBeams, 5 = LightsOff, 6 = Unlock, 7 = Lock, 8 = Trunk, 9 = Cluster text «NOCT», 10 = Doors Unlock Interior, 11 = Doors Lock Key.
- **0x80** — запуск цикличного светового шоу (моргалки по I-Bus).
- **0x81** — остановка светового шоу.

Сервис UUID: `1a2b0001-5e6f-4a5b-8c9d-0e1f2a3b4c5d`, характеристика для записи: `1a2b0002-5e6f-4a5b-8c9d-0e1f2a3b4c5d`. Для отправки можно использовать любое BLE-приложение с возможностью записи в характеристику (nRF Connect, LightBlue и т.п.).

### OBD и сдвиговая лампа

При включённом в `config.h` опросе OBD (`NOCT_OBD_ENABLED=1`) и подключённом ELM327 по UART (пины в config) на экране отображаются RPM и температуры (ОЖ, масло). При RPM ≥ 5500 встроенный LED моргает, а на приборную панель (IKE) периодически отправляется текст «SHIFT!».

Подробно: [BMW_E39_Assistant.md](bmw/BMW_E39_Assistant.md).

## 7. Режим Forza

**Вход:** Меню → **Monitoring** → **Forza** (долгое нажатие).

- Дашборд телеметрии из игры Forza (UDP, порт 5300). Нужно включить в игре Data Out и указать IP устройства. На экране: скорость, передача, RPM, shift lamp при достижении порога.

Подробно: [FORZA_SETUP.md](forza/FORZA_SETUP.md).

## 8. Режимы Hacker (WiFi/BLE)

**Вход:** Меню → **Hacker** → выбор пункта (WiFi Clone, BLE Clone, BLE Spam, EAPOL Capture) → долгое нажатие.

- **WiFi Clone:** сначала режим RADAR (список сетей). Дождитесь окончания скана («SCANNING…» сменится на список). Короткое нажатие — выбор сети, долгое при выбранной сети — запуск клона (Evil Twin AP + captive portal). Если долгое нажатие при незавершённом скане или без выбранной сети — тост «Scan…» или «Select network». На экране TRAP отображаются пароль точки и логи. При ошибке запуска AP показывается «AP failed».
- **BLE Clone:** сканирование BLE; короткое — выбор устройства в списке, долгое — клонировать (реклама под выбранным именем). Список в едином стиле с меню, имена и RSSI.
- **BLE Spam:** спам BLE-рекламы; счётчик пакетов.
- **EAPOL Capture:** пассивный перехват EAPOL-кадров (WPA/WPA2 handshake) для последующего подбора пароля (например, в aircrack-ng). Список AP с пометкой [EAPOL]. Только для авторизованного тестирования своих сетей.

Во всех этих режимах подсказка внизу подсказывает жесты (1x next, 2s clone, 2x menu и т.д.). Выход: двойной тап → меню → двойной тап.

## 9. Настройки (Config)

**Вход:** Меню → **Config** → выбор пункта.

Настройки сохраняются в NVS (переживают перезагрузку).

| Пункт | Описание |
|-------|----------|
| **AUTO** | Карусель сцен: OFF → 5 s → 10 s → 15 s → OFF. |
| **FLIP** | Поворот дисплея на 180°. |
| **GLITCH** | Включение/выключение визуального глитча. |
| **LED** | Включение/выключение подсветки LED при алертах. |
| **DIM** | Низкая яркость по умолчанию при включении. |
| **CONTRAST** | Цикл уровней контраста. |
| **TIMEOUT** | Таймаут затемнения дисплея: 0 / 30 / 60 с. |

Долгое нажатие на пункте — переключение варианта или вход в подменю (зависит от пункта).

## 10. Система (System)

**Вход:** Меню → **System** → выбор пункта.

| Пункт | Действие |
|-------|----------|
| **REBOOT** | Перезагрузка устройства (с подтверждением). |
| **Charge only** | Режим зарядки (см. выше). |
| **POWER OFF** | Глубокий сон (потребление минимум). Пробуждение — по кнопке (в зависимости от платы). |
| **VERSION** | Показ версии прошивки (NOCTURNE_VERSION из config.h). |

## 11. Частые проблемы и решения

| Проблема | Что проверить |
|----------|----------------|
| Меню не открывается | Двойной тап должен быть быстрым (два нажатия подряд). Убедитесь, что не удерживаете кнопку слишком долго. |
| Нет данных в мониторинге (NO SIGNAL) | Wi‑Fi в `secrets.h`, устройство в той же сети, что и ПК. Запущен `monitor.py`, порт в `config.json` = `TCP_PORT` в `secrets.h`. Firewall не блокирует порт. |
| Нет связи по I-Bus (BMW) | В `config.h`: `NOCT_IBUS_ENABLED=1`, проводка TX 39 / RX 38, трансивер (MCP2004A/TH3122), подключение к шине (разъём CD-чейнджера или магнитола). |
| Телефон не подключается к «BMW E39 Key» | Режим BMW Assistant активен; при необходимости перезагрузка платы; сброс списка BLE-устройств на телефоне. |
| Экран пустой или артефакты | Проверка питания OLED (Vext GPIO 36 — LOW), I2C (SDA 17, SCL 18, RST 21). В config.h — правильные пины. |
| Батарея не показывается / неверный % | Калибровка в config.h: `NOCT_BAT_DIVIDER_FACTOR`, `NOCT_VOLT_MIN`/`NOCT_VOLT_MAX`, `NOCT_BAT_CALIBRATION_OFFSET`. Проводка ADC (GPIO 1) и управления делителем (GPIO 37). |

Дополнительно: разделы «Устранение неполадок» в [PC_MONITORING.md](monitoring/PC_MONITORING.md) и [BMW_E39_Assistant.md](bmw/BMW_E39_Assistant.md).
