# BMW E39 Assistant — подключение и настройка

Модуль **BMW E39 Assistant** в Nocturne OS добавляет для BMW с I-Bus/K-Bus функции: BLE-ключ по телефону, I-Bus (замки, свет, MFL, PDC, приборка), вывод трека на OLED, заглушки OBD и сдвиговая лампа.

---

## Поддерживаемые платформы

| Платформа | Поддержка | Примечание |
|-----------|-----------|------------|
| **BMW E39** (дорест) | Основная | Полностью тестируемая цель; I-Bus/K-Bus 9600 8E1. |
| **BMW E46** | Совместимость по I-Bus | Те же идентификаторы устройств и сообщений (I-Bus/K-Bus); при необходимости уточнить формат пакетов PDC/MFL под конкретный год. |
| **BMW E38** | Ожидаемая совместимость | Общая шина I-Bus/K-Bus того же формата; проверка на реальном автомобиле рекомендуется. |
| **BMW E53** | Ожидаемая совместимость | Аналогично E38/E46. |
| **Электроника не старше MARK 4** | Ожидаемая совместимость | При сохранении протокола I-Bus/K-Bus (9600 8E1) и тех же адресов устройств (GM, CDC, IKE, LCM, PDC, MFL и т.д.) модуль может использоваться; при расширении поддержки — тесты и при необходимости доп. парсинг сообщений. |

---

## Кратко

1. **Вход в режим:** меню (двойной тап) → **BMW** → **BMW Assistant** (долгое нажатие).
2. **Управление на плате:** короткое нажатие — следующее действие в списке; долгое (~2 s) — выполнить выбранную команду по I-Bus (при отсутствии синха с шиной показывается «No IBus»).
3. **Выход:** двойной тап → меню → с уровня категорий снова двойной тап.
4. **Без I-Bus:** если шина не подключена или отключена в конфиге (`NOCT_IBUS_ENABLED=0`), отображается «No IBus»; BLE-ключ и список действий на экране работают.
5. **Управление с телефона:** приложение на Android может подключаться по BLE к «BMW E39 Key» и отправлять команды (свет, замки, light show, Now Playing). Протокол и UUID описаны в [BMW_ANDROID_APP.md](BMW_ANDROID_APP.md).

## 1. Назначение

| Функция | Описание |
|--------|----------|
| **Подход = открыть, отход = закрыть** | Плата в режиме BMW Assistant рекламирует BLE с именем «BMW E39 Key». Подключение телефона → отправка команды разблокировки по I-Bus; отключение (с задержкой ~2.5 с) → блокировка. |
| **Температуры и диагностика** | Зарезервирован API OBD: `setObdData()`, отображение на экране. Для работы нужен ELM327 (UART или BLE) и запрос PIDs (температура ОЖ, масла, RPM). |
| **Shift lamp** | При подключённом OBD и RPM ≥ 5500 встроенный LED моргает (как в режиме Forza). |
| **Мультимедиа** | Парсинг кнопок руля (MFL) по I-Bus → действия Next/Prev/Play-Pause/Vol. Вывод названия трека и исполнителя на OLED (поля задаются через `setNowPlaying()`). Звук с телефона через плату (A2DP Sink) — опционально, с внешним I2S DAC. |
| **Управление светом** | Отправка команд по I-Bus: провожающий свет (GoodbyeLights), Follow Me Home, парковочные огни, аварийка и др. через `sendGoodbyeLights()`, `sendFollowMeHome()`, `sendParkLights()`, `sendHazardLights()`. |
| **Парктроники (PDC)** | Приём сообщений от модуля PDC (0x60) по I-Bus, отображение дистанций на OLED. |
| **Текст на приборку** | Отправка текста на комбинацию приборов (IKE) через `sendClusterText()` (формат IKE_TXT_GONG). |

Дополнительно: при `NOCT_IBUS_MONITOR_VERBOSE=1` все пакеты I-Bus логируются в Serial (hex) для отладки и реверса. Парсинг IKE (температура по сообщению 0x19) реализован; значение доступно через `getIkeCoolantC()` при отсутствии OBD. Одометр/скорость по I-Bus и эмуляция CDC для вывода на дисплей магнитолы можно добавить по тому же принципу.

## 2. Требования

- Плата **Heltec WiFi LoRa 32 V4** (ESP32-S3).
- Автомобиль **BMW E39** (дорест) или совместимый по I-Bus/K-Bus (**E46**, **E38**, **E53** и др., см. таблицу выше).
- Смартфон с BLE для работы «ключом».
- Для I-Bus: **трансивер** (MCP2004A, TH3122 или оптопара) — шина не 3.3 V логика.
- Для OBD (по желанию): адаптер ELM327 (UART или BLE).
- Для качественного звука с телефона (по желанию): внешний I2S DAC (например PCM5102) и AUX на магнитоле.

## 3. Подключение

### 3.1 Распиновка Heltec V4 (используемые выводы)

| Назначение | GPIO | Примечание |
|------------|------|------------|
| I-Bus TX   | 39   | NOCT_IBUS_TX_PIN |
| I-Bus RX   | 38   | NOCT_IBUS_RX_PIN |
| Кнопка     | 0    | PRG, меню / выход |
| LED        | 35   | Shift lamp, алерты |
| Питание    | 5 V / 3.3 V | USB или внешнее |
| Земля      | GND  | Общая с кузовом |

OLED (SDA 17, SCL 18, RST 21) и батарея уже используются прошивкой.

### 3.2 Подключение к I-Bus

- I-Bus/K-Bus: **9600 8E1**, однопроводная шина. Напряжения не TTL — нужен трансивер (MCP2004A, TH3122 или схема на оптопаре, см. [I-K_Bus](https://github.com/muki01/I-K_Bus) и схемы в документации).
- В E39 шину удобно подключать:
  - разъём CD-чейнджера в багажнике (если проводка есть);
  - или за магнитолой / в районе блока предохранителей.
- Цвет провода I-Bus (ориентировочно): белый/красный с жёлтыми точками. Минус — коричневый, плюс 12 V — с другого разъёма при необходимости.

**Схема подключения:** Плата (GPIO 39 TX, GPIO 38 RX, GND) → трансивер (MCP2004A/TH3122: TTL ↔ шина) → I-Bus автомобиля (один провод + масса). Земля платы обязательно с кузовом.

### 3.3 Питание

- Рекомендуется питание от прикуривателя или постоянный плюс (через реле зажигания при необходимости). Минус — надёжно на кузов.

### 3.4 OBD2 (опционально)

- В прошивке реализован клиент ELM327 по UART (Serial2). В **`include/nocturne/config.h`** задать `NOCT_OBD_ENABLED 1` и пины `NOCT_OBD_TX_PIN` / `NOCT_OBD_RX_PIN` (по умолчанию 9 и 10). Адаптер ELM327 подключается к этим пинам (38400 8N1). Прошивка запрашивает PIDs 01 0C (RPM), 01 05 (температура ОЖ), 01 5C (температура масла) и передаёт данные в BmwManager; они отображаются на экране BMW Assistant и используются для shift-лампы (LED + текст «SHIFT!» на IKE).

### 3.5 Аудио: звук с телефона через плату (A2DP Sink + I2S DAC)

При включённом в **config.h** флаге **NOCT_A2DP_SINK_ENABLED 1** плата в режиме BMW Assistant работает как **A2DP Sink**: телефон подключается к устройству «BMW E39 Audio» по Bluetooth (как к колонке/наушникам), аудио передаётся на плату и выводится по **I2S** на внешний DAC.

**Подключение:**

1. **Пины I2S** (по умолчанию, см. config.h): **BCK = GPIO 45**, **WS (LRCK) = GPIO 46**, **DOUT (данные) = GPIO 40**. Питание 3.3 V и GND — с платы Heltec (Ve и GND).
2. **Внешний I2S DAC** (например **PCM5102A**, **UDA1334A**):
   - BCK → BCK платы (GPIO 45)  
   - LRCK/WS → WS платы (GPIO 46)  
   - DIN/DATA → DOUT платы (GPIO 40)  
   - VCC → 3.3 V (Ve), GND → общий.
3. **Выход DAC** (аналог L/R или объединённый) → вход **AUX** головного устройства или отдельного усилителя.

Итог: звук с телефона по Bluetooth → плата Heltec V4 → I2S → DAC → AUX в машину. Отдельная «звуковая» плата не нужна. Даташит платы: **DataSheets/WiFi_LoRa_32_V4.2.0.pdf** (свободные выводы J3).

## 4. Прошивка и настройка

1. **Сборка**  
   В PlatformIO: окружение `heltec_wifi_lora_32_V4`, команда сборки из корня проекта.

2. **Прошивка**  
   Загрузить прошивку через USB (порт выбран в среде).

3. **Вход в режим BMW Assistant**  
   Двойной тап по кнопке (PRG) → меню → категория **BMW** → **BMW Assistant** (долгое нажатие для входа).

4. **Сопряжение телефона**  
   Включить Bluetooth на телефоне, найти устройство **«BMW E39 Key»**, подключиться. После подключения плата отправит по I-Bus команду разблокировки (если шина уже в синхе). При отходе и потере связи через ~2.5 с — команда блокировки.

5. **Настройки**  
   Порог RPM для shift lamp задаётся в коде (сейчас 5500). Включение/выключение OBD, медиа, PDC — через соответствующие вызовы API и (при добавлении) пункты меню.

## 5. Использование

- **Замки:** подойти с телефоном (подключился BLE) — разблокировка; отойти (связь пропала) — блокировка.
- **Температуры/OBD:** при включённом `NOCT_OBD_ENABLED=1` модуль ObdClient опрашивает ELM327 по UART (пины `NOCT_OBD_TX_PIN` / `NOCT_OBD_RX_PIN`) и передаёт данные в `bmwManager.setObdData()`; RPM и температуры выводятся на экран BMW Assistant. Без OBD можно опираться на данные с шины (IKE, см. ниже).
- **Кнопки руля:** нажатия MFL парсятся, последнее действие доступно как `getLastMflAction()` (Next/Prev/Play-Pause/Vol). Дальше — маппинг в AVRCP или HID на телефоне (доработка).
- **Трек на OLED:** вызов `bmwManager.setNowPlaying("Track Name", "Artist")` (из приложения или при появлении AVRCP «Now Playing»).
- **Управление с платы:** в режиме BMW Assistant на экране отображается текущее выбранное действие. **Короткое нажатие** — следующее действие (Goodbye, FollowMe, Park, Hazard, LowBeam, LightsOff, Unlock, Lock, Trunk, Cluster, **DoorUnlk**, **DoorLock**). **Долгое нажатие** — выполнить выбранную команду по I-Bus (при отсутствии синха с шиной показывается «No IBus»). Двойной тап — выход в меню.
- **Управление светом и замками с телефона (BLE):** при подключении к «BMW E39 Key» доступна GATT-характеристика для записи. Один байт 0–11 = команда (0=GoodbyeLights … 9=Cluster, 10=DoorUnlk, 11=DoorLock). Байт **0x80** — запуск цикличного светового шоу (Hazard → Park → Goodbye → LowBeam → LightsOff, интервал 800 мс). Байт **0x81** — остановка шоу. Сервис: `1a2b0001-5e6f-4a5b-8c9d-0e1f2a3b4c5d`, характеристика: `1a2b0002-5e6f-4a5b-8c9d-0e1f2a3b4c5d`. Подойдут приложения nRF Connect, LightBlue и т.п.
- **Опрос шины:** раз в 3 с отправляется запрос статуса IKE (опрос I-Bus), чтобы шина оставалась активной.
- **Свет (с платы, с телефона по BLE или из кода):** GoodbyeLights, FollowMeHome, ParkLights, HazardLights, LowBeams, LightsOff, Lock/Unlock, Trunk, текст на приборку (Cluster), DoorUnlk/DoorLock.
- **Парктроники:** при появлении сообщений PDC (0x60) дистанции выводятся на OLED.
- **Текст на приборку:** `bmwManager.sendClusterText("HELLO")` — отправить строку на комбинацию (до ~20 символов, кодировка OEM).
- **Shift на приборке:** при подключённом OBD и RPM ≥ 5500 раз в секунду на IKE отправляется текст «SHIFT!» (плюс моргание LED на плате).

## 6. Устранение неполадок

| Проблема | Что проверить |
|----------|----------------|
| Нет связи по I-Bus (статус «IBUS --») | Проводка TX/RX, питание трансивера, правильность точки подключения к шине (CD-чейнджер/магнитола), 9600 8E1. |
| Замки не реагируют | Подключение телефона к «BMW E39 Key»; что плата в режиме BMW Assistant; что I-Bus уже в синхе (статус «IBUS OK»). |
| Телефон не подключается | Отключить другие BLE-режимы (WiFi в режиме BMW выключен); перезапуск платы; сброс списка BLE на телефоне. |
| OBD не отвечает | В config.h включён ли `NOCT_OBD_ENABLED`, правильные пины TX/RX; ELM327 на 38400 8N1; питание адаптера. Ответы ELM327 в формате с пробелами (например «41 0C 12 34») предпочтительны. |
| PDC не показывается | Наличие модуля PDC в машине и сообщений 0x60 на шине; при необходимости уточнить формат пакета под свою модель. |
| Команды с телефона не срабатывают | Убедиться, что записывается байт в нужную характеристику (`1a2b0002-...`); I-Bus в синхе (на экране «IBUS OK»). Команды 0x80/0x81 не требуют синха для старта/остановки шоу, но отправка по шине — только при синхе. |

## 7. Проверка работоспособности

- **I-Bus:** включить в **config.h** `NOCT_IBUS_MONITOR_VERBOSE 1`, прошить плату, подключить USB, открыть монитор порта (115200 бод). При нажатии кнопок руля (MFL), включении задней передачи (PDC) и т.д. в логе должны появляться строки пакетов в hex. Так проверяют, что шина жива и трансивер подключён верно.
- **Команды с платы:** в режиме BMW Assistant выбрать действие (свет/замки и т.д.) и выполнить долгим нажатием — на экране при синхе с I-Bus отображается выполнение; при отсутствии синха — «No IBus».
- **Команды с телефона:** подключиться по BLE к «BMW E39 Key», отправить команду (через приложение или nRF Connect в характеристику 1a2b0002) — при синхе с I-Bus должна сработать соответствующая команда (свет, замки и т.д.).

## 8. Отладка и полный дамп шины

- **Логирование всех пакетов I-Bus:** в **`include/nocturne/config.h`** задать `NOCT_IBUS_MONITOR_VERBOSE 1`. Каждый принятый пакет будет выводиться в Serial в hex (источник, длина, данные). Скорость Serial 115200. Используется для реверса протокола и отладки.
- **Температура с шины (IKE):** при приёме сообщения IKE с командой 0x19 (температура) значение сохраняется и доступно через API `getIkeCoolantC()`. Может использоваться при отсутствии OBD для отображения температуры ОЖ с шины.
