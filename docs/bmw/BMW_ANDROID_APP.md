# BMW Assistant — протокол BLE для Android-приложения

Описание BLE GATT-сервисов и команд для управления модулем **BMW E39 Assistant** (Nocturne OS) с Android-телефона.

---

## Подключение

- **Имя устройства (advertising name):** `BMW E39 Key`
- **Сервис управления (UUID):** `1a2b0001-5e6f-4a5b-8c9d-0e1f2a3b4c5d`

Подключитесь к устройству по имени или отфильтруйте сканирование по этому UUID сервиса.

**Автоматическое открытие/закрытие:** при установлении GATT-подключения прошивка отправляет команду разблокировки по I-Bus; при отключении (с задержкой ~2.5 с) — блокировку. Отдельно отправлять Unlock/Lock при подключении не требуется.

## Характеристики

### 1. Команды (WRITE)

| UUID характеристики | Свойства | Описание |
|---------------------|----------|----------|
| `1a2b0002-5e6f-4a5b-8c9d-0e1f2a3b4c5d` | WRITE | Один байт — код команды (см. таблицу ниже). |

**Таблица команд (1 байт):**

| Байт (hex) | Действие |
|------------|----------|
| 0x00 | Goodbye Lights (провожающий свет) |
| 0x01 | Follow Me Home |
| 0x02 | Park Lights (парковочные огни) |
| 0x03 | Hazard Lights (аварийка) |
| 0x04 | Low Beams (ближний свет) |
| 0x05 | Lights Off (выключить свет) |
| 0x06 | Unlock (разблокировка) |
| 0x07 | Lock (блокировка) |
| 0x08 | Trunk Open (открыть багажник) |
| 0x09 | Cluster Text "NOCT" (текст на приборку) |
| 0x0A | Doors Unlock Interior (разблокировка дверей изнутри) |
| 0x0B | Doors Lock Key (блокировка ключом) |
| 0x80 | Start Light Show (запуск цикличной «моргалки») |
| 0x81 | Stop Light Show (остановка light show) |

Команды 0x00–0x0B выполняются только при синхронизации с I-Bus («IBus OK»). Команды 0x80/0x81 также требуют I-Bus.

### 2. Статус (READ / NOTIFY)

| UUID характеристики | Свойства | Описание |
|---------------------|----------|----------|
| `1a2b0003-5e6f-4a5b-8c9d-0e1f2a3b4c5d` | READ, NOTIFY | Пакет статуса: флаги I-Bus, PDC, OBD, температуры (см. формат ниже). |

**Формат пакета статуса (10 байт):**

| Смещение | Размер | Описание |
|----------|--------|----------|
| 0 | 1 | Флаги: bit0 = ibus_sync (1 = I-Bus OK), bit1 = phone_connected, bit2 = pdc_valid, bit3 = obd_connected |
| 1 | 1 | Coolant temp (°C), 0xFF = нет данных |
| 2 | 1 | Oil temp (°C), 0xFF = нет данных |
| 3 | 2 | RPM (big-endian), 0xFFFF = нет данных |
| 5 | 4 | PDC distances (см): FL, FR, RL, RR (по 1 байту), 0xFF = нет данных |
| 9 | 1 | Last MFL action: 0 = none, 1 = next, 2 = prev, 3 = play_pause, 4 = vol_up, 5 = vol_down |

Прошивка обновляет характеристику при изменении; приложение может подписаться на NOTIFY и/или читать по запросу (READ).

### 3. Now Playing (WRITE)

| UUID характеристики | Свойства | Описание |
|---------------------|----------|----------|
| `1a2b0004-5e6f-4a5b-8c9d-0e1f2a3b4c5d` | WRITE | Трек и исполнитель для вывода на OLED. Формат: UTF-8 строка `track\0artist` (null между названием трека и исполнителем). Макс. длина 96 байт суммарно. |

Пример: записать байты `"My Song\0Artist Name"` — на дисплее платы отобразится трек и исполнитель. При синхе с I-Bus тот же текст отправляется на дисплей магнитолы (MID) через UPDATE_MID.

### 4. Текст на приборку (WRITE)

| UUID характеристики | Свойства | Описание |
|---------------------|----------|----------|
| `1a2b0005-5e6f-4a5b-8c9d-0e1f2a3b4c5d` | WRITE | Произвольный текст на комбинацию приборов (IKE). UTF-8 строка до 20 байт. При синхе с I-Bus отправляется sendClusterText(). |

Пример: записать `"NOCT"` или `"HI"` — текст появится на приборной панели (IKE_TXT_GONG).

## Минимальная реализация приложения

1. Сканировать BLE по имени «BMW E39 Key» или по UUID сервиса `1a2b0001-5e6f-4a5b-8c9d-0e1f2a3b4c5d`.
2. Подключиться к GATT-серверу.
3. Найти сервис `1a2b0001-5e6f-4a5b-8c9d-0e1f2a3b4c5d`.
4. Для отправки команды: записать 1 байт (0x00–0x0B, 0x80, 0x81) в характеристику `1a2b0002-5e6f-4a5b-8c9d-0e1f2a3b4c5d`.
5. Подписаться на NOTIFY характеристики статуса `1a2b0003-...` и парсить 10 байт (флаги, coolant, oil, RPM, PDC, last MFL).
6. Записать в характеристику Now Playing `1a2b0004-...` строку `track\0artist` для обновления вывода на OLED и на магнитолу (MID).
7. Опционально: записать в характеристику текста на приборку `1a2b0005-...` строку до 20 байт UTF-8 для вывода на IKE.

Разрешения Android: `BLUETOOTH_SCAN`, `BLUETOOTH_CONNECT`, `ACCESS_FINE_LOCATION` (для BLE-сканирования на Android 12+).

## Подключение по USB (опционально)

При включённой в прошивке опции **USB CDC** (`NOCT_USB_CDC_ENABLED=1` и поддержка платой) плату можно подключить к телефону по кабелю **USB-OTG** (телефон — хост). Устройство появится как виртуальный последовательный порт. В приложении Android используйте USB Host API (`UsbManager`, `UsbDeviceConnection`, `UsbSerialDriver` и т.п.) и тот же протокол команд: отправка одного байта (0x00–0x0B, 0x80, 0x81) равносильна записи в BLE-характеристику команд. Разрешения: `android.hardware.usb.host` и при необходимости объявление `usb-device` в манифесте. Подробнее см. [HELTEC_V4_BOARD_AND_CONFIG.md](../board/HELTEC_V4_BOARD_AND_CONFIG.md).
